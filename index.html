一、Markdown 语法

Markdown Preview Enhanced 是一款为 Atom 以及 Visual Studio Code 编辑器编写的超级强大的 Markdown 插件。

一、基础语法

1、分级标题

（1）类 atx 形式

   ## 二级标题

   #### 四级标题

   ##### 五级标题

   ###### 六级标题

（2）类 Setext 形式

   一级标题
   ============

   二级标题
   ------------

2.2分割线

在一行中用三个以上的星号(*)、减号(-)、底线(_)来建立一个分隔线，行内不能有其他内容,也可以在星号或是减号中间插入空格，效果相同。

代码：

*******
-------
_______
* * *

3、列表

3.1有序列表

使用“X.”（X代指数字）表示其后是一个有序列表，“.”后需追加空格。

代码：

1. 选项一
2. 选项二
3. 选项三

效果：

1.选项一

2.选项二

3.选项三

3.2无序列表

使用“-”、“+”、“*”（三者功能相同）表示其后是一个列表，符号后同样需要追加空格。

代码：

- 题目一
- 题目二
- 题目三

效果：

  • 题目一
  • 题目二
  • 题目三

3.3待办事项

使用“[ ]”表示一个方框(一对方括号间有一个空格)，使用“[X]”表示一个被勾选的方框（x不区分大小写）。

代码：

内嵌HTML：
 - [ ] 吃早饭
 - [X] 跑步
 - [ ] 学习
-[]空格：每个符号间都要有空格
-[]吃早饭
-[X]跑步
-[]学习

效果：

  • 吃早饭
  • [X] 跑步
  • 学习

- 1. 题目一

3.5列表与引用、代码块、标题、段落混合使用

①定义型列表：定义型列表由名词和解释组成。一行写上定义，紧跟一行写上解释。解释的写法:紧跟一个缩进。

②列表缩进：列表项目标记通常是放在最左边，但是其实也可以缩进，最多 3 个空格，项目标记后面则一定要接着至少一个空格或制表符。

③包含段落的列表：列表项目可以包含多个段落，每个项目下的段落都必须缩进 4 个空格或是 1 个制表符。

④包含引用的列表：在列表项目内放进引用，那 > 就需要缩进

⑤包含代码区块的引用：如果要放代码区块的话，该区块就需要缩进两次，也就是 8 个空格或是 2 个制表符。

3.6注意事项

在使用列表时，只要是数字后面加上英文的点，就会无意间产生列表，比如2017.12.30 表达的是日期，有些软件把它被误认为是列表。解决方式：在每个点前面加上\就可以了。

代码：

2019\.8\.10

效果：

2019.8.10
